TASK 3 — TRAILING STOP FIX (REPLACES highest_price_since_entry model)
=============================================================

The QC algo trailing stop works like this:
  Every bar:

    stop_loss = max(existing_stop_loss, new_stop)  # ratchet up, never down

This means the stop is driven by LIVE ATR × 3 behind the CURRENT price,
not by historical peak. The stop only moves up, never down.

A) In check_exits() for LONG positions, REPLACE the trailing stop calculation:

   REPLACE WITH:
     # QC algo: stop = current_price - (live_ATR × 3), ratcheted up with max()
     new_stop = current_close - (live_atr * TRAILING_STOP_ATR_MULT)

     # Retrieve the stored stop_loss from the open signal (persisted in signals table)
     active_sigs_for_stop = get_active_signals(strategy_name=STRATEGY_NAME, asset=asset)
     stored_stop = None
     if active_sigs_for_stop:
         stored_stop = active_sigs_for_stop[0].get("stop_loss")

     # Ratchet: trailing stop only moves in favour of the trade
     if stored_stop is not None:
         trailing_stop = max(stored_stop, new_stop)   # LONG: higher stop is better
     else:
         trailing_stop = new_stop

     # Persist updated stop_loss back to the signals table if it moved
     if stored_stop is None or trailing_stop > stored_stop:
         for sig in active_sigs_for_stop:
             update_signal_stop_loss(sig["id"], trailing_stop)
         logger.info(
             f"[TREND-NONFX-EXIT] Position #{pos_id} | "
             f"Trailing stop updated: {stored_stop} → {trailing_stop:.5f} "
             f"(price={current_close:.5f} - {TRAILING_STOP_ATR_MULT}×ATR={live_atr:.6f})"
         )
     else:
         logger.info(
             f"[TREND-NONFX-EXIT] Position #{pos_id} | "
             f"Trailing stop held at {trailing_stop:.5f} "
             f"(new_stop={new_stop:.5f} would have moved stop down — not allowed)"
         )
