Add a persistent asset_class label to every trading signal so users
can filter signals by stocks, crypto, forex, commodities, or indices.
The asset_class is determined at insert time and stored in the DB.

Files to change: models.py, database.py, api_v1.py
Do not change any strategy file, any other API file, or any other
function beyond those explicitly listed below.

Step E: After the migration runs, add a one-time backfill that
sets asset_class on existing signals where it is NULL or 'other'
and the asset is in _ASSET_CLASS_MAP. Add this function:

  def _backfill_asset_class():
      """One-time backfill: set asset_class on existing signals
      that predate the column. Safe to run on every startup —
      only updates rows where asset_class is NULL or 'other'
      and the asset has a known mapping.
      """
      with _get_session() as session:
          try:
              rows = session.query(Signal).filter(
                  Signal.asset_class.in_([None, "other"])
              ).all()
              updated = 0
              for sig in rows:
                  mapped = _ASSET_CLASS_MAP.get(sig.asset)
                  if mapped and mapped != "other":
                      sig.asset_class = mapped
                      updated += 1
              if updated:
                  session.commit()
                  logger.info(
                      f"[DB] Backfilled asset_class for "
                      f"{updated} existing signal(s)"
                  )
              else:
                  logger.info(
                      "[DB] asset_class backfill: all signals "
                      "already classified"
                  )
          except Exception as e:
              session.rollback()
              logger.error(f"[DB] asset_class backfill failed: {e}")

  Then call _backfill_asset_class() inside init_db() after
  _migrate_schema() is called:

  FIND in init_db():
    _migrate_schema()

  Add after it:
    _backfill_asset_class()
