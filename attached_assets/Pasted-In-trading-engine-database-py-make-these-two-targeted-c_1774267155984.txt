In trading_engine/database.py make these two targeted changes only.

1. In seed_strategy_assets(), find the reactivation block inside 
   the seed loop. It currently reactivates any row where 
   is_active=0. Change it to only reactivate rows where added_by 
   equals "system_seed" — never reactivate rows that were manually 
   removed by an admin (added_by != "system_seed"):

   FIND this pattern inside the seed loop:
     if not existing:
         session.add(StrategyAsset(...))
         seeded += 1
     # some variants also have:
     elif not existing.is_active:
         existing.is_active = 1
         ...
         session.commit()

   CHANGE the reactivation condition to:
     if not existing:
         session.add(StrategyAsset(
             strategy_name=strategy_name,
             symbol=symbol,
             asset_class=asset_class,
             sub_category=sub_cat,
             is_active=1,
             fcsapi_verified=1,
             added_by="system_seed",
         ))
         seeded += 1
     elif not existing.is_active and existing.added_by == "system_seed":
         existing.is_active = 1
         existing.sub_category = sub_cat
         session.commit()
         logger.info(
             f"[DB] seed_strategy_assets: reactivated "
             f"{strategy_name}/{symbol} (system seed)"
         )

   This means: only auto-reactivate rows that were originally 
   seeded by the system. If an admin removed it (added_by is 
   anything other than system_seed or is_active was set to 0 
   by admin action), leave it deactivated.
