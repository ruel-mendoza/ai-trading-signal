=============================================================
CONTEXT — WHAT THE QC ALGORITHM DOES
=============================================================

The source QuantConnect algorithm implements a trend-following strategy with these rules:

LONG entry:
  - price >= 50-day highest close (breakout)
  - SMA(50) > SMA(100)
  - Apply to: equities, crypto, commodities (long only)

SHORT entry:
  - price <= 50-day lowest close (breakdown)
  - SMA(50) < SMA(100)
  - Apply to: forex pairs only in QC — in our system map this to a SHORT_ELIGIBLE subset

Position sizing:
  - Risk 1% of portfolio value per trade
  - quantity = (portfolio_value × 0.01) / (3 × ATR)

Trailing stop:
  - stop_distance = 3 × ATR (in QC this is live ATR; in our system keep ATR fixed at entry)
  - For LONG: trailing_stop = peak_price_since_entry − (3 × ATR_at_entry)
  - For SHORT: trailing_stop = trough_price_since_entry + (3 × ATR_at_entry)
  - Exit triggered when closing price hits trailing stop (we use closing-rule gate at 4:01 PM ET)

50-day LOW tracking:
  - Track lowest close of prior 50 days (same as highest_50d but for the low side)
  - SHORT entry fires when current_close <= lowest_50d AND sma50 < sma100

The QC algorithm sizes positions as: quantity = (portfolio_value × 0.01) / (3 × ATR)

B) Add a helper function in trend_non_forex.py:
```python
def _calculate_quantity(portfolio_value: float, atr: float, atr_mult: float = TRAILING_STOP_ATR_MULT) -> Optional[float]:
    """
    QC algo: quantity = (portfolio_value × RISK_PCT) / (atr_mult × ATR)
    Returns None if ATR is zero or portfolio_value is not provided.
    """
    stop_distance = atr_mult * atr
    if stop_distance <= 0 or portfolio_value <= 0:
        return None
    return round((portfolio_value * RISK_PCT_PER_TRADE) / stop_distance, 4)
```
