Add a "Backfill" button for each strategy and a "Last Run" timestamp display in the System Status tab, under the Signal Integrity widget in admin.py.

## What to build

### 1. New API endpoint — POST /admin/api/backfill/strategy

Add this endpoint to admin.py that triggers a backfill for a given strategy:

- Accept JSON body: { "strategy_name": "mtf_ema", "timeframe": "D1" } (timeframe optional, defaults to all timeframes for that strategy)
- Run cache.force_refresh() for every active asset in the strategy using the same ASSET_TIMEFRAMES mapping already defined in backfill.py
- Store the result in app_settings DB table with key "backfill_last_run_{strategy_name}" as a JSON string: { "ran_at": "ISO timestamp", "succeeded": N, "failed": N, "failed_symbols": [] }
- Return JSON: { "success": true, "strategy": "mtf_ema", "succeeded": N, "failed": N, "failed_symbols": [], "ran_at": "ISO timestamp" }
- Use the engine_registry to get the existing cache instance — do NOT create a new FCSAPIClient

The ASSET_TIMEFRAMES mapping for each strategy is:
  mtf_ema:           ["D1", "4H", "1H"]
  trend_forex:       ["D1"]
  trend_non_forex:   ["D1"]
  sp500_momentum:    ["30m", "D1"]
  highest_lowest_fx: ["1H", "D1"]
  stocks_algo1:      ["D1"]
  stocks_algo2:      ["D1"]

### 2. New API endpoint — GET /admin/api/backfill/status

Returns last run info for all strategies:

- Read all "backfill_last_run_{strategy_name}" keys from app_settings
- Return JSON: { "strategies": { "mtf_ema": { "ran_at": "...", "succeeded": N, "failed": N }, ... } }
- For strategies with no record yet, return null for that key

### 3. Update the Signal Integrity widget in admin.py (_build_signal_integrity or equivalent inline HTML)

Below the existing "Run Diagnostic" button, add a new section titled "Data Backfill" with:

- A card/row for each of the 7 strategies showing:
  - Strategy display name (same labels as STRATEGY_LABELS in api_v1.py)
  - Last run timestamp — "Never" if no record, otherwise formatted as "Apr 2, 2026 01:04 UTC"
  - Last run result — green "97 ok" / red "3 failed" badges if available
  - A "Backfill" button that calls the POST endpoint for that strategy
  - A small loading spinner (CSS only, toggle via JS) while the request is in flight
  - On success: update the last run timestamp and result inline without page reload
  - On failure: show a red error message inline

### 4. JavaScript (add to ADMIN_JS in admin.py)

Add these functions:

async function loadBackfillStatus()
- Calls GET /admin/api/backfill/status
- Populates the last run timestamp and result badges for each strategy row
- Called on page load when tab is "system" and after each backfill completes

async function runBackfill(strategyName, btnEl)
- Disables the button and shows a spinner
- POSTs to /admin/api/backfill/strategy with { "strategy_name": strategyName }
- On success: updates the row inline, shows green success toast using existing showVerifyToast()
- On error: shows red error toast
- Re-enables the button after completion
- Calls loadBackfillStatus() after completion to refresh all rows

### 5. Style requirements

- Match the existing dark theme (background #0f172a, borders #334155, text #e2e8f0)
- Use the same badge classes already defined in ADMIN_CSS: badge status-active for success, badge status-closed for failure
- Loading spinner: pure CSS, inline, 16px, uses border + border-top trick, animate spin 1s linear infinite
- Add @keyframes spin to ADMIN_CSS
- Backfill button style: same as existing btn-verify class but blue tint (rgba(59,130,246,0.12), color #3b82f6)

### 6. Guard rails

- The backfill endpoint must be admin-role only (_admin_role_guard)
- If the engine is not available (engine_registry.get_engine() returns None), return 503 with message "Engine not ready — start the application first"
- Cap maximum symbols per strategy at 150 to prevent accidental credit drain — return 400 if strategy has more than 150 active assets
- Each backfill run should log a single INFO line per strategy: "[ADMIN-BACKFILL] strategy=mtf_ema | succeeded=11 failed=0 | ran_at=..."
- Do NOT touch any existing endpoints, CSS classes, or JS functions — only add new ones
