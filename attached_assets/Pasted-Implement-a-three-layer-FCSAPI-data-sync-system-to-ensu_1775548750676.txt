Layer 2 — Pre-signal entry price validation
Add a new function _validate_entry_price() to trading_engine/cache_layer.py:
pythondef _validate_entry_price(
    self,
    asset: str,
    cached_price: float,
    timeframe: str = "D1",
    tolerance_pct: float = 0.5,
) -> dict:
    """
    Cross-check the cached candle close price against a fresh FCSAPI
    advance endpoint call before allowing a signal to be inserted.

    If the live price differs from the cached price by more than
    tolerance_pct, the entry is flagged as invalid and a candle
    refresh is triggered automatically.

    tolerance_pct defaults:
      forex pairs  → 0.5%
      crypto       → 2.0%  (higher volatility)
      ETFs/stocks  → 1.0%

    Returns:
      {
          "valid": True,
          "cached_price": 139.16,
          "live_price": 138.94,
          "diff_pct": 0.16,
          "tolerance_pct": 0.5,
          "refreshed": False,
          "reason": "within_tolerance"
      }
    """
The function should:

Call self.api_client.get_advance_data([asset], period="1d", merge="latest") to get the live price
Calculate diff_pct = abs(live_price - cached_price) / cached_price * 100
Use asset-class-aware tolerance:

Crypto (/USD pairs): 2.0%
ETFs (no / in symbol): 1.0%
Forex pairs: 0.5%


If diff_pct > tolerance_pct:

Log a warning with both prices and the difference
Call self.force_refresh(asset, timeframe) to update the cache
Return valid=False with reason="price_discrepancy"


If advance API is unavailable, return valid=True with reason="advance_unavailable_skipped" — do not block the signal on API failure
Log result clearly:

   [PRICE-VALID] USO/D1 | cached=139.16 live=138.94 diff=0.16% <= 1.0% | VALID
   [PRICE-VALID] USO/D1 | cached=139.16 live=135.00 diff=3.0% > 1.0% | INVALID — refreshing cache

python# ── Startup candle sync — ensure all assets have fresh data ─────────────────
async def _startup_candle_sync():
    try:
        from trading_engine.strategies.multi_timeframe import get_all_mtf_assets
        from trading_engine.strategies.trend_non_forex import get_active_symbols as _tnf
        from trading_engine.strategies.trend_forex import get_active_symbols as _tf

        pairs = []
        for sym in get_all_mtf_assets():
            pairs.extend([(sym, "D1"), (sym, "4H"), (sym, "1H")])
        for sym in _tnf():
            pairs.append((sym, "D1"))
        for sym in _tf():
            pairs.append((sym, "D1"))
        pairs.append(("SPX", "30m"))
        pairs.append(("SPX", "D1"))

        # Deduplicate
        pairs = list(dict.fromkeys(pairs))

        logger.info(f"[STARTUP-SYNC] Syncing {len(pairs)} asset/timeframe pairs...")
        result = cache.ensure_fresh_candles_batch(pairs)
        logger.info(
            f"[STARTUP-SYNC] Complete | "
            f"refreshed={result['refreshed']} | "
            f"fresh={result['already_fresh']} | "
            f"failed={result['failed']}"
        )
    except Exception as e:
        logger.warning(f"[STARTUP-SYNC] Failed (non-fatal): {e}")
