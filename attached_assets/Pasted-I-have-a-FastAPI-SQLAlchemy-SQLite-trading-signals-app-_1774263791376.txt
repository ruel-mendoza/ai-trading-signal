PROBLEM:
has_any_open_signal_for_asset() checks ALL strategies and blocks any strategy 
from entering if ANY other strategy has an open signal on the same asset. This 
causes:
  1. mtf_ema and sp500_momentum block each other on SPX
  2. highest_lowest_fx and trend_forex block each other on EUR/USD

SOLUTION:
Add an optional exclude_strategies parameter to has_any_open_signal_for_asset() 
in database.py, then update each strategy to exclude its known non-conflicting 
partner strategies from the cross-strategy block.

Make ALL of the following changes in ONE pass:

1. trading_engine/database.py

Update has_any_open_signal_for_asset() to accept an optional 
exclude_strategies parameter. When provided, signals from those strategies 
are ignored in the cross-strategy check. All other behavior stays exactly 
the same:

  def has_any_open_signal_for_asset(
      asset: str,
      exclude_strategies: Optional[list[str]] = None,
  ) -> bool:
      with _get_session() as session:
          q = (
              session.query(
                  Signal.id,
                  Signal.strategy_name,
                  Signal.direction,
              )
              .filter(
                  Signal.asset == asset,
                  Signal.status == "OPEN",
              )
          )
          if exclude_strategies:
              q = q.filter(
                  Signal.strategy_name.notin_(exclude_strategies)
              )
          row = q.first()
          if row:
              logger.debug(
                  f"[DB] has_any_open_signal_for_asset({asset}): "
                  f"blocked by signal #{row[0]} | "
                  f"strategy={row[1]} | direction={row[2]}"
              )
              return True
          return False
